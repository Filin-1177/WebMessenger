# 🌐 LAN Messager

Локальный мессенджер с веб-интерфейсом, работающий по принципу клиент-сервер. Поддерживает текстовые сообщения, передачу файлов, историю чатов и уведомления в реальном времени. Работает исключительно в локальной сети (LAN).

## ✨ Особенности

- **Веб-клиент**: Не требует установки ПО, доступ через браузер.
- **Real-time**: Мгновенная доставка сообщений через WebSocket.
- **Файлы**: Отправка изображений и документов.
- **История**: Сохранение всех сообщений на сервере (SQLite).
- **Уведомления**: Всплывающие уведомления о новых сообщениях.
- **Безопасность**: Работает только внутри вашей локальной сети.
- **Markdown**: Поддержка форматирования текста (жирный, курсив, код и т.д.).

## 🚀 Быстрый старт

### 1. Требования
- Python 3.8+
- Библиотеки: `aiohttp`, `aiosqlite`

Установка зависимостей:
```bash
pip install aiohttp aiosqlite
```

### 2. Запуск сервера
Сервер автоматически определяет свой IP и запускает веб-интерфейс.

```bash
python server.py
```

После запуска в консоли появится адрес для подключения (например: `http://192.168.1.5:8080`).

### 3. Подключение клиентов
Откройте указанный адрес в любом браузере на устройствах в той же сети.
- При первом входе придумайте имя пользователя.
- Разрешите уведомления в браузере, чтобы не пропускать сообщения.

## ⚙️ Функционал

| Компонент | Описание |
|-----------|----------|
| **Чат** | Общий канал для всех пользователей. |
| **Файлы** | Drag & Drop или кнопка загрузки. |
| **Админка** | Управление пользователями (кик/бан) доступна в интерфейсе сервера. |
| **Настройки** | Смена темы (светлая/темная), управление подключением. |

## 📂 Структура проекта

```text
LAN_Messager/
├── server.py          # Основной файл сервера (API + Web + DB)
├── database.db        # База данных (создается автоматически)
├── uploads/           # Папка для загруженных файлов
└── README.md          # Инструкция
```

## 🛠️ Технологии

- **Backend**: Python (`asyncio`, `aiohttp`)
- **Database**: SQLite (`aiosqlite`)
- **Frontend**: HTML5, CSS3, Vanilla JavaScript
- **Protocol**: WebSocket (для чата), HTTP (для файлов)

## ⚠️ Примечания

- При перезапуске сервера история сообщений сохраняется.
- Для корректной работы файлы должны сохраняться в папку `uploads`.
- Если вы меняете сеть (Wi-Fi), IP-адрес сервера может измениться.
